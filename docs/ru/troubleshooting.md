# Troubleshooting

Типовые проблемы при работе с `HapticTrace` и базовые проверки.

## phyphox

### `Connect Sensors` ничего не делает

Проверьте:

- Mac и сенсорное устройство в одной сети
- URL указан полностью, включая порт
- в `phyphox` включен remote access
- эксперимент `phyphox` запущен

### Ошибки `502`, `connection refused`, `max retries exceeded`

Обычно причина в одном из следующих пунктов:

- неверный URL
- эксперимент не запущен
- устройство недоступно по сети
- endpoint remote access больше невалиден

### На графике нет сенсорных данных

Проверьте:

- в эксперименте включены нужные сенсоры
- измерение действительно идет
- выбран режим с сенсорами
- подключение к сенсорам выполнено успешно

## iPhone / iPad capture

### `Connect iPhone` не находит устройство

Проверьте:

- USB-подключение
- доверие между устройством и Mac
- видит ли macOS устройство как источник захвата
- не занят ли capture device другим приложением

### Нет live preview

Проверьте:

- в `System Settings > Privacy & Security > Camera` разрешен доступ для `HapticTrace`
- устройство переподключено
- другие приложения, использующие capture stack, закрыты
- подключение заново открыто из `HapticTrace`
- выбран режим с видео
- runtime-лог `~/Library/Logs/HapticTrace/HapticTrace.log` содержит строку `First iPhone video frame received`

### Видео не начинает записываться

Проверьте:

- preview уже выдал хотя бы один кадр
- iPhone/iPad подключен
- выбран режим с видео
- системные разрешения macOS не блокируют захват

### Приложение запускается, но захват не начинается

Обычно причина в одном из следующих пунктов:

- нет системных разрешений macOS
- для устройства недоступен capture stack
- macOS некорректно видит устройство

Если окно показывает `connected; waiting for video frames`, соединение с устройством создано, но первый кадр еще не пришел. В этом случае сначала проверьте разрешение Camera для `HapticTrace`, затем runtime-лог `~/Library/Logs/HapticTrace/HapticTrace.log`.

## Синхронизация

### Видео и сенсоры заметно рассинхронизированы

Это допустимо, так как видео и сенсоры могут приходить с разных устройств.

Что делать:

- открыть `Offset Settings`
- скорректировать `manual offset`
- выровнять запись по экранному событию или выраженному пику сигнала

### Курсор или playback работают некорректно

Проверьте:

- выполнен `Reset View`
- заново задан нужный zoom range
- текущая сессия содержит валидные данные

## Сохранение и загрузка

### Сессия не сохраняется

Проверьте:

- директория доступна для записи
- на диске есть свободное место
- сессия содержит данные
- путь autosave или ручного сохранения валиден

### Сессия не загружается

Проверьте:

- файл является корректной `.zip`-сессией
- архив не поврежден
- архив содержит ожидаемые данные
- в системе доступен нужный playback stack

## Базовые проверки

Перед разбором проблем проверьте:

- выбран правильный режим записи
- нужные устройства подключены
- URL `phyphox` указан корректно
- у macOS есть нужные разрешения
- текущее состояние сессии соответствует сценарию работы

## Примечания

- `sensor path` и `video path` независимы и могут ломаться отдельно
- `automatic offset` — это стартовая оценка, а не точная синхронизация
