# Использование

Этот документ описывает работу с `HapticTrace` для записи, синхронизации и анализа сессий с хаптическим откликом.

## Обзор

`HapticTrace` поддерживает три режима записи:

- **`Sensors + Video`** — синхронная запись сенсорных данных и screen capture
- **`Sensors Only`** — захват сенсорных данных без видео
- **`Video Only`** — запись экрана без сенсорных данных

Основной сценарий — `Sensors + Video`: сенсорные данные записываются с устройства с `phyphox`, а экранный контекст и воспроизведение хаптиков — с целевого iPhone/iPad.

## Рекомендуемая конфигурация

Типовая синхронная конфигурация включает:

- **сенсорное устройство** с `phyphox`, в котором включены `accelerometer` и `gyroscope`
- **целевой iPhone/iPad**, на котором запускается тестируемое приложение
- **Mac** с запущенным `HapticTrace`

Для стабильных измерений оба мобильных устройства должны быть жестко зафиксированы относительно друг друга на протяжении всей записи. Любое относительное смещение добавляет паразитный сенсорный сигнал.

## Сценарий записи

### 1. Подготовьте сенсорное устройство

На сенсорном устройстве:

1. Откройте `phyphox`.
2. Создайте или загрузите эксперимент, в котором включены:
   - `accelerometer`
   - `gyroscope`
3. Включите remote access.
4. Скопируйте URL эксперимента.

### 2. Подготовьте целевое устройство

1. Подключите целевой iPhone/iPad к Mac через USB.
2. Убедитесь, что macOS определяет его как источник захвата экрана.
3. Запустите на целевом устройстве тестируемое приложение.

### 3. Подключите устройства в HapticTrace

1. Запустите `HapticTrace`.
2. Откройте `Connection -> Connection Settings`.
3. Выберите нужный **Recording mode**.
4. Укажите URL `phyphox`, если выбранный режим использует сенсоры.
5. Нажмите **Connect Sensors**, если выбранный режим использует сенсоры.
6. Нажмите **Connect iPhone**, если выбранный режим использует видео.

Когда оба источника данных доступны, приложение готово к записи.

### 4. Выполните запись

1. Нажмите **Play** в главном окне.
2. Воспроизведите нужный хаптический сценарий в тестируемом приложении.
3. Нажмите **Stop** после завершения записи.

После `Stop` сессия становится доступной для playback и анализа.

## Основной сценарий анализа

После записи типовой сценарий работы выглядит так:

1. просмотреть измеренный сигнал
2. при необходимости проанализировать спектрограмму
3. воспроизвести записанное видео
4. проверить синхронизацию между сигналом и событиями на экране
5. при необходимости скорректировать offset
6. сохранить сессию для повторного анализа или сравнения

## Главное окно

Главное окно — основной рабочий интерфейс для анализа сигнала.

Оно включает:

- вкладки сессий, если открыто несколько сессий
- кнопки управления, такие как `Play`, `Stop`, `Clear`, `Compare`, `Reset View`
- отображение сигнала
- опциональную спектрограмму
- переключатели отображения сигнала, такие как `Accelerometer`, `Gyroscope`, `Envelope`
- стандартные инструменты matplotlib для zoom и pan

Главное окно используется для просмотра сигнала, увеличения интересующих участков и перемещения playback cursor по timeline.

## Окно Video Playback

Окно `Video Playback` используется и для preview, и для воспроизведения записанного видео.

До начала записи оно показывает live preview с подключенного iPhone/iPad.  
После записи переключается в режим playback и показывает записанное видео на общей временной шкале.

Основные элементы управления:

- `Play/Pause`
- `Restart`
- `Step -`
- `Step +`
- скорость воспроизведения
- `Offset Settings`

В режиме `Sensors Only` окно playback не используется.

## Окно Connection

Окно `Connection` используется для настройки пайплайна записи.

Оно предоставляет:

- выбор режима записи
- поле URL `phyphox`
- элементы подключения сенсоров
- элементы подключения iPhone/iPad
- статус подключения и записи

Это окно используется при переключении между `Sensors + Video`, `Sensors Only` и `Video Only`, а также при переподключении устройств.

## Окно Log

Окно `Log` содержит информацию о состоянии выполнения:

- статус подключения
- статус записи
- сообщения о сохранении и загрузке
- ошибки и диагностические события

Используйте его, когда нужно понять текущее состояние сессии или разобраться с неожиданным поведением во время обычной работы.

## Playback и синхронизация

После записи `HapticTrace` синхронизирует отображение сигнала, спектрограммы и записанного видео на общей временной шкале.

### Поведение курсора

- video timeline и курсор на графике связаны
- перемещение курсора обновляет текущую позицию анализа
- двойной клик по графику или спектрограмме перемещает playback в выбранное время

### Playback по видимому диапазону

Если график сигнала зазумен, playback следует текущему видимому диапазону времени. Это удобно для анализа короткого события или одного хаптического импульса.

### Reset View

`Reset View` восстанавливает полный временной диапазон и сбрасывает текущее состояние zoom/pan.

## Подстройка offset

Синхронизация состоит из двух частей:

- **automatic offset** — начальная автоматически оцененная синхронизация
- **manual offset** — пользовательская поправка поверх автоматической оценки

Для точной подстройки откройте **Offset Settings** из окна playback.

Типовые случаи использования:

- хаптическое событие визуально происходит немного раньше или позже относительно видео
- нужно совместить конкретное экранное событие с измеренным пиком сигнала
- автоматическая синхронизация близка к нужной, но недостаточно точна для анализа

Offset можно менять вживую, поэтому результат сразу виден и на видео, и на графиках.

## Спектрограмма

Спектрограмма показывает сигнал в частотной области.

Она полезна, когда нужно:

- проанализировать распределение энергии хаптического события
- сравнить импульсы с разным спектральным содержимым
- изучить короткие переходные процессы, которые хуже видны во временной области

### Настройки спектрограммы

В `Tools -> Spectrogram Settings` настраиваются:

- оконная функция
- display interpolation
- анализируемый фрагмент
- режим последних секунд
- максимальная отображаемая частота
- cleanup mode
- `nperseg`
- overlap ratio
- `nfft` multiplier

Эти параметры влияют на качество анализа, но не изменяют исходные записанные данные.

## Интерполяция

`Tools -> Interpolation Settings` управляет derived-интерполяцией обработанного сигнала для аналитических задач.

Это полезно, когда нужна более плотная форма сигнала для:

- построения спектрограммы
- более стабильного анализа на сильном увеличении
- экспорта на основе интерполированного представления

Интерполяция используется как аналитический слой. Исходный обработанный сигнал остается без изменений.

Доступные параметры включают:

- включение и отключение интерполяции
- target samples on window
- длину окна
- overlap ratio
- оконную функцию
- метод интерполяции
- порядок полинома
- post smoothing
- apply to export

## Работа с сессиями

### Новые и текущие сессии

При запуске приложения автоматически создается новая пустая сессия.

Важно:

- запись начинается только в пустой сессии
- `Clear` влияет только на активную сессию
- переключение вкладок сохраняет состояние каждой сессии
- `Compare` имеет смысл, когда открыто несколько сессий

### Save Session

`Save Session` сохраняет текущую сессию в один `.zip`-архив.

Сессия может включать:

- сенсорные данные
- обработанный сигнал
- графики
- записанное видео
- metadata сессии
- metadata синхронизации

Такой формат делает сессию переносимой и удобной для повторного открытия.

### Load Session

`Load Session` восстанавливает ранее сохраненную `.zip`-сессию.

Поведение:

- если текущая сессия пустая, архив загружается в нее
- если текущая сессия уже содержит данные, создается новая вкладка сессии

### Rename Session

Используйте `Edit -> Rename Session`, чтобы переименовать текущую сессию.

### Autosave

Если приложение закрывается при наличии данных сессии, autosave сохраняет архив в настроенную директорию автосохранения.

## Compare mode

`Compare` предназначен для анализа нескольких сессий.

Он позволяет сравнивать сессии в разных представлениях:

- сравнение сигналов
- сравнение спектрограмм
- overlay view
- split view

Типовые сценарии:

- сравнение разных хаптических паттернов
- сравнение одного и того же сценария между сборками
- анализ различий по таймингу или интенсивности

## Сохранение результатов

Помимо сохранения и загрузки полной сессии, приложение позволяет сохранять отдельные результаты:

- **Save Graphs** — экспорт изображений графиков
- **Save Video** — экспорт записанного видеофайла

Эти команды полезны, когда нужен отдельный артефакт без сохранения всей сессии.

## Управление

### Основная область графика

- двойной клик по графику или спектрограмме — переместить playback в выбранное время
- колесо мыши — zoom
- перетаскивание левой кнопкой мыши — pan
- `Reset View` — восстановить полный вид

### Video playback

- `Space` — play/pause
- `Left` — шаг назад
- `Right` — шаг вперед

## Обзор меню

### File

Используется для жизненного цикла сессии и экспорта:

- новая сессия
- закрытие сессии
- сохранение и загрузка сессии
- сохранение графиков
- сохранение видео
- выход

### Edit

- переименование текущей сессии

### Connection

Используется для подключения и отключения сенсоров и целевого iPhone/iPad.

### View

Используется для управления видимостью окна playback, спектрограммы и окна логов.

### Tools

Используется для расширенных настроек анализа сигнала, таких как спектрограмма и интерполяция.

## Примечания

- идеальная аппаратная синхронизация не предполагается, так как сенсоры и видео могут приходить с разных устройств
- automatic offset — это стартовая оценка, а не точная гарантия совпадения
- для точного анализа ручная подстройка часто необходима
- старые записи, полученные через legacy video pipeline, могут воспроизводиться иначе
